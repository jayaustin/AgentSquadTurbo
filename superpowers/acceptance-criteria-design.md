# Acceptance Criteria Design

## Overview

Convert vague intent into work that can be implemented and validated without
guessing.

## Required Output

- state the goal, non-goals, and relevant constraints
- define the expected behavior or artifact clearly
- call out important edge cases and failure conditions
- make success measurable with concrete acceptance checks
- note assumptions, dependencies, and open questions

## Anti-Patterns

- requirements that depend on unstated context
- "done" criteria that are subjective or untestable
- hidden scope expansions
- handoffs that omit edge cases or failure behavior
