# Risk-Based Prioritization

## Overview

Prioritize work by actual project risk, not by whichever item is easiest to talk
about.

## Prioritize Using

- user or business impact
- likelihood of failure or defect
- dependency criticality
- reversibility and rollback cost
- time sensitivity or release exposure

## Anti-Patterns

- treating all findings as equally urgent
- prioritizing by effort alone
- hiding tradeoffs between speed and safety
- deferring high-risk work without explicit acceptance
